# Pizza-Sales-Analysis-SQL-Queries
Description:
This project demonstrates my proficiency in SQL for data analysis by performing a comprehensive analysis of a pizza sales dataset. The analysis focuses on extracting business insights, identifying trends, and preparing actionable reports.

Key Highlights / Skills Demonstrated:
SQL Querying: SELECT, JOIN, GROUP BY, HAVING, ORDER BY, Subqueries, CTEs
Aggregation & Metrics: Total sales, revenue per pizza, quantity sold, top-selling and low-performing products
Sorting & Filtering: Identifying bottom 5 and top 5 pizza items by revenue and quantity
Analytical Thinking: Translating raw sales data into meaningful insights for business decisions
Reporting Skills: Structured query results suitable for dashboards or MIS reporting

Learning Outcome:
Strengthened hands-on SQL skills for data extraction and reporting
Applied analytical logic to identify trends, bottlenecks, and revenue opportunities
Prepared insights in a recruiter-friendly format demonstrating practical business impact

Use Case:
This project simulates a real-world retail analytics scenario and can be used as a reference for business intelligence reporting, sales trend analysis, and KPI monitoring.

Tools:
SQL (MySQL / PostgreSQL / SQLite)
CSV or sample dataset for Pizza Sales

Ideal For:
Entry-level Data Analysts, Junior SQL Analysts, BI enthusiasts looking to demonstrate practical SQL problem-solving skills.
